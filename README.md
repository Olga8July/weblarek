# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

### Данные
Приложение работает с тремя источниками данных:

- Серверное API — основной источник данных о товарах. Запросы выполняются через класс `Api` к эндпоинту `/products` методом `GET`. Ответ сервера содержит массив объектов с полями: `id`, `title`, `description`, `image`, `category`, `price`.

- Локальное состояние — данные о состоянии корзины и оформлении заказа хранятся во временной памяти приложения (в моделях). При перезагрузке страницы состояние сбрасывается.

- Переменные окружения — базовый URL API задаётся через переменную `VITE_API_ORIGIN` из файла `.env`. Это позволяет легко переключать окружения (разработка/продакшен) без изменения кода.

Все данные проходят строгую типизацию через интерфейсы из `src/types/index.ts`, что гарантирует безопасность работы с данными на этапе разработки.

#### Основные интерфейсы
```typescript
// Товар
interface IProduct {
  id: string;           // Уникальный идентификатор товара
  description: string;  // Описание товара
  image: string;        // URL изображения товара
  title: string;        // Название товара
  category: string;     // Категория товара
  price: number | null; // Цена товара
} 

// Покупатель (данные для заказа)
interface IBuyer {
  payment: 'card' | 'cash'; // Способ оплаты: карта или наличные
  email: string;            // Email покупателя
  phone: string;            // Телефон покупателя
  address: string;          // Адрес доставки
}
```
### Модели данных
В приложении реализованы три класса-модели, каждый из которых отвечает за свою зону ответственности в хранении и управлении данными.

#### Класс ProductCatalog
Отвечает за хранение и управление каталогом товаров. Хранит все доступные товары и текущий выбранный товар для подробного просмотра.

Конструктор:  
`constructor()` — создаёт пустой каталог товаров.

Поля класса:  
`items: IProduct[]` — массив всех товаров в каталоге.  
`selectedItem: IProduct | null` — текущий выбранный товар для отображения (или `null`, если ничего не выбрано).

Методы класса:  
`setProducts(products: IProduct[]): void` — сохраняет массив товаров, полученный от сервера.
`getProducts(): IProduct[]` — возвращает массив всех товаров в каталоге.  
`getProductById(id: string): IProduct | undefined` — возвращает товар по его идентификатору или `undefined`, если товар не найден.  
`setSelectedItem(product: IProduct): void` — устанавливает товар для подробного отображения.  
`getSelectedItem(): IProduct | null` — возвращает текущий выбранный товар или `null`.

#### Класс Cart
Отвечает за хранение и управление товарами в корзине покупателя. Хранит массив выбранных товаров и предоставляет методы для работы с корзиной.

Конструктор:  
`constructor()` — создаёт пустую корзину.

Поля класса:  
`items: IProduct[]` — массив товаров, выбранных покупателем для покупки.

Методы класса:  
`getItems(): IProduct[]` — возвращает массив товаров, которые находятся в корзине.  
`addItem(product: IProduct): void` — добавляет товар, полученный в параметре, в массив корзины.  
`removeItem(productId: string): void` — удаляет товар из массива корзины по его идентификатору.  
`clear(): void` — очищает корзину (удаляет все товары из массива).  
`getTotal(): number` — возвращает стоимость всех товаров в корзине (сумма цен всех товаров).  
`getCount(): number` — возвращает количество товаров в корзине.  
`hasItem(productId: string): boolean` — проверяет наличие товара в корзине по его идентификатору, полученному в параметре метода. Возвращает `true`, если товар найден, иначе `false`.

#### Класс Buyer
Отвечает за хранение и валидацию данных покупателя при оформлении заказа. Хранит информацию о способе оплаты, контактах и адресе доставки.

Конструктор:  
`constructor()` — создаёт пустой объект данных покупателя.

Поля класса:  
`payment: 'card' | 'cash' | null` — способ оплаты (`'card'` — карта, `'cash'` — наличные, `null` — не выбран).  
`address: string` — адрес доставки.  
`email: string` — email покупателя.  
`phone: string` — телефон покупателя.

Методы класса:  
`setField(field: string, value: string): void` — устанавливает значение для указанного поля. Позволяет обновлять отдельные поля без сброса остальных данных  
`setPayment(payment: 'card' | 'cash'): void` — устанавливает способ оплаты  
`setAddress(address: string): void` — устанавливает адрес доставки  
`setEmail(email: string): void` — устанавливает email  
`setPhone(phone: string): void` — устанавливает телефон  
`getData(): IBuyer` — возвращает объект со всеми данными покупателя  
`clear(): void` — очищает все данные покупателя (сбрасывает в начальное состояние)  
`validate(): { valid: boolean; errors: Record<string, string> }` — выполняет валидацию всех полей. Возвращает объект с флагом `valid` (общая валидность) и объектом `errors` с текстами ошибок для невалидных полей. Пример результата:  
```typescript
{
  valid: false,
  errors: {
    payment: 'Не выбран вид оплаты',
    email: 'Укажите емэйл'
  }
}
```

### Слой коммуникации
Класс слоя коммуникации отвечает за взаимодействие приложения с сервером. Он использует композицию с базовым классом `Api` для выполнения HTTP-запросов.

#### Класс ApiClient
Данный класс выполняет запрос на сервер с помощью метода `GET` класса Api и получает с сервера объект с массивом товаров, а также с помощью метода `POST`класса Api отправляет на сервер данные о покупателе и выбранных товарах.

Конструктор:  
`constructor(api: IApi)` — принимает экземпляр класса `Api`, реализующего интерфейс `IApi`.

Поля класса:  
`api: IApi` — экземпляр для выполнения запросов к серверу.

Методы класса:  
`getProducts(): Promise<IServerProductsResponse>` — выполняет GET запрос на эндпоинт `/product/` и возвращает объект с массивом товаров.
`createOrder(order: IOrderRequest): Promise<IOrderResponse>` — выполняет POST запрос на эндпоинт `/order/` с данными о заказе и возвращает подтверждение покупки.

### Слой Views (Представление)
В приложении реализованы компоненты, отвечающие за отображение данных и взаимодействие с пользователем. Все компоненты наследуются от базового класса Component и управляются через события.

#### Класс Basket
Отвечает за отображение содержимого корзины покупателя: списка товаров, общей стоимости и кнопки оформления заказа.

Конструктор:
`constructor(container: HTMLElement, events: IEvents)` — принимает DOM-контейнер шаблона корзины и объект для работы с событиями.

Поля класса:
`basketListElement: HTMLElement` — элемент списка товаров `(.basket__list)`.
`orderButton: HTMLButtonElement` — кнопка оформления заказа `(.basket__button)`.
`totalPrice: HTMLElement` — элемент для отображения итоговой суммы `(.basket__price)`.

Интерфейс данных (IBasket):
`basketList: HTMLElement[]` — массив DOM-элементов карточек товаров в корзине.
`basketPrice: number` — общая стоимость всех товаров в корзине.
`buttonStatus: boolean` — состояние кнопки «Оформить»: `true` — неактивна (корзина пуста), `false` — активна.

Методы класса:
`set basketList(items: HTMLElement[]): void` — заменяет содержимое списка товаров на переданный массив элементов.
`set basketPrice(value: number): void` — устанавливает текст итоговой суммы в формате `${value}` синапсов.
`set buttonStatus(value: boolean): void` — устанавливает атрибут `disabled` кнопки в зависимости от значения (`true` — кнопка заблокирована).

Поведение:
При клике на кнопку «Оформить» эмитится событие `'basket:order'`, которое инициирует процесс оформления заказа. Компонент не хранит состояние — все данные передаются извне через метод `render()`.

#### Класс Card
Базовый компонент для отображения карточек товаров. Содержит общую логику отображения названия и цены, которая используется во всех представлениях карточек (каталог, превью, корзина).

Конструктор:
`constructor(container: HTMLElement)` — принимает DOM-контейнер шаблона карточки.

Поля класса:
`titleElement: HTMLElement` — элемент заголовка товара `(.card__title)`.
`priceElement: HTMLSpanElement` — элемент отображения цены `(.card__price)`.

Интерфейс данных `(ICard)`:
`title: string` — название товара.
`price: number | null` — цена товара; если `null`, отображается как «Бесценно».

Методы класса:
`set title(value: string): void` — устанавливает текст заголовка карточки.
`set price(value: number | null): void` — устанавливает отображаемую цену:
 • если `value !== null` — показывает `${value}` синапсов,
 • если `value === null` — показывает «Бесценно».

Поведение:
Компонент является абстрактной базой для специализированных карточек (`CardCatalog, CardPreview, CardBasket`). Он не используется напрямую, а наследуется подклассами, которые добавляют собственную разметку и логику (изображение, категория, кнопки и т.д.). Все данные передаются извне через метод `render()`, состояние не хранится.

#### Класс CardBasket
Компонент отображает карточку товара внутри корзины покупателя. Наследуется от базового класса `Card` и расширяет его функциональностью: отображением порядкового номера товара и кнопкой удаления из корзины.

Конструктор:
`constructor(container: HTMLElement, actions: ICardActions)` — принимает DOM-контейнер шаблона карточки и объект с обработчиком клика по кнопке удаления.

Поля класса:
`cardIndex: HTMLElement` — элемент для отображения порядкового номера товара `(.basket__item-index)`.
`itemDeleteButton: HTMLButtonElement` — кнопка удаления товара из корзины `(.basket__item-delete)`.

Интерфейс данных `(ICardBasket)`:
Наследует все поля от `ICard (title, price)` и добавляет:
`itemIndex: number` — порядковый номер товара в списке корзины.

Методы класса:
`set itemIndex(value: number): void` — устанавливает текст порядкового номера в соответствующий элемент.

Поведение:
При клике на кнопку «Удалить» вызывается обработчик `actions.onClick`, который эмитит событие `'basket::delete'` с передачей данных о товаре. Компонент не хранит состояние — все данные `(title, price, itemIndex)` передаются извне через метод `render()`.

#### Класс CardCatalog
Компонент отображает карточку товара в каталоге. Наследуется от базового класса `Card` и расширяет его функциональностью: отображением изображения, категории и поддержкой клика по всей карточке для просмотра деталей.

Конструктор:
`constructor(container: HTMLElement, actions?: ICardActions)` — принимает DOM-контейнер шаблона карточки и необязательный объект с обработчиком клика.

Поля класса:
`categoryElement: HTMLElement` — элемент отображения категории товара `(.card__category)`.
`imageElement: HTMLImageElement` — элемент изображения товара `(.card__image)`.

Интерфейс данных `(ICardCatalog)`:
Наследует все поля от `ICard (title, price)` и добавляет:
`category: string` — категория товара (например, «софт», «другое»),
`image: string` — URL изображения товара.

Методы класса:
`set category(value: string): void` — устанавливает текст категории и применяет соответствующий CSS-класс из categoryMap для цветовой индикации.
`set image(value: string): void` — устанавливает изображение с помощью утилитарного метода setImage, используя заголовок карточки как альтернативный текст (alt).

Поведение:
При клике на карточку (если передан обработчик `actions.onClick`) эмитится событие `'card::select'`, которое инициирует открытие модального окна с подробной информацией о товаре. Компонент не хранит состояние — все данные `(title, price, category, image)` передаются извне через метод `render()`.

#### Класс CardPreview
Компонент отображает подробную информацию о товаре в модальном окне предварительного просмотра. Наследуется от базового класса `Card` и расширяет его функциональностью: отображением изображения, категории, описания и интерактивной кнопки для добавления/удаления товара из корзины.

Конструктор:
`constructor(container: HTMLElement, events: IEvents)` — принимает DOM-контейнер шаблона карточки и объект для работы с событиями.

Поля класса:
`imageElement: HTMLImageElement` — элемент изображения товара `(.card__image)`.
`categoryElement: HTMLSpanElement` — элемент отображения категории `(.card__category)`.
`textElement: HTMLParagraphElement` — элемент описания товара `(.card__text)`.
`addButton: HTMLButtonElement` — кнопка управления корзиной `(.card__button)`.

Интерфейс данных `(ICardPreview)`:
Наследует все поля от `ICard (title, price)` и добавляет:
`category: string` — категория товара,
`image: string` — имя файла изображения,
`text: string` — описание товара,
`buttonText: string` — текст на кнопке («Купить» / «Удалить из корзины» / «Недоступно»),
`buttonDisabledState: boolean` — состояние кнопки (`true` — неактивна).

Методы класса:
`set category(value: string): void` — устанавливает текст категории и применяет соответствующий CSS-класс из categoryMap для цветовой индикации.
`set image(value: string): void` — устанавливает полный путь к изображению `(/src/images/${value})` с заголовком в качестве альтернативного текста.
`set buttonText(value: string): void` — устанавливает текст на кнопке.
`set buttonDisabledState(state: boolean): void` — блокирует или разблокирует кнопку в зависимости от значения.

Поведение:
При клике на кнопку эмитится событие `'preview::to-cart'`, которое обрабатывается презентером для добавления или удаления товара из корзины. Состояние кнопки и её текст динамически меняются в зависимости от наличия товара в корзине и доступности цены. Компонент не хранит состояние — все данные передаются извне через метод `render()`.

#### Класс Form
Базовый компонент для работы с формами. Предоставляет общую функциональность отображения ошибок валидации и служит основой для всех специализированных форм приложения (оформление заказа, контактная информация).

Конструктор:
`constructor(container: HTMLFormElement)` — принимает DOM-элемент формы.

Поля класса:
`formErrors: HTMLSpanElement` — элемент для отображения сообщений об ошибках `(.form__errors)`.

Интерфейс данных `(IForm)`:
`errors: string` — текст ошибки или пустая строка, если ошибок нет.

Методы класса:
`set errors(value: string): void` — устанавливает текст ошибки в соответствующий элемент. Если передана пустая строка, ошибки не отображаются.

Поведение:
Компонент является абстрактной базой для подклассов `(FormOrder, FormContacts)`, которые добавляют собственные поля и логику. Он не используется напрямую, а наследуется для обеспечения единообразного отображения ошибок валидации по всему приложению. Состояние не хранится — данные передаются извне через метод `render()`.

#### Класс FormOrder
Компонент отображает форму выбора способа оплаты и адреса доставки на первом шаге оформления заказа. Наследуется от базового класса Form и обеспечивает взаимодействие с пользователем через кнопки выбора оплаты, поле ввода адреса и кнопку перехода к следующему шагу.

`cashButton: HTMLButtonElement` — кнопка выбора оплаты наличными `(button[name=cash])`.
`addressInput: HTMLInputElement` — поле ввода адреса доставки `(input[name=address])`.
`nextButton: HTMLButtonElement` — кнопка «Продолжить» `(.order__button)`.

### События приложения
Взаимодействие между слоями реализовано через централизованную систему событий на основе EventEmitter. Все события разделены по функциональным зонам.

Каталог товаров
`'catalog:changed'` — эмитится после загрузки товаров с сервера; инициирует рендеринг галереи.
`'card::select'` — эмитится при клике на карточку товара в каталоге; передаёт выбранный товар.
`'catalog::select'` — эмитится при открытии детального просмотра; использует текущий выбранный товар из модели.

Управление корзиной
`'preview::to-cart'` — эмитится при клике на кнопку в модальном окне предпросмотра; добавляет или удаляет товар из корзины.
`'cart::change'` — эмитится при любом изменении содержимого корзины; обновляет отображение корзины и счётчик в шапке.
`'basket::open'` — эмитится при клике на иконку корзины в шапке; открывает модальное окно с содержимым корзины.
`'basket::delete'` — эмитится при клике на кнопку удаления в карточке корзины; передаёт товар для удаления.
`'basket::make-order'` — эмитится при клике на кнопку «Оформить» в корзине; инициирует первый шаг оформления заказа.

Оформление заказа
`'form::card' / 'form::cash'` — эмитятся при выборе способа оплаты; обновляют модель покупателя.
`'form::address', 'form::email', 'form::phone'` — эмитятся при изменении полей форм; передают текущее значение.
`'form::next'` — эмитится при клике на «Продолжить» в форме адреса; открывает форму контактных данных.
`'form::submit'` — эмитится при отправке формы контактов; инициирует отправку заказа на сервер.
`'buyer::set-data'` — эмитится при изменении данных покупателя; обновляет состояние форм и отображает ошибки валидации.

Завершение покупки
`'success::close'` — эмитится при закрытии экрана подтверждения; скрывает модальное окно.

Каждое событие содержит только необходимые данные, что обеспечивает слабую связанность компонентов и упрощает поддержку кода.
